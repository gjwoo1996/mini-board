# Mini-Board 프론트엔드 코드 분석

## 기술 스택
- Next.js 16 (App Router)
- React 19
- Tailwind CSS 4
- TypeScript

## 디렉터리 구조

```
frontend/
├── app/                    # App Router 페이지
│   ├── layout.tsx          # 루트 레이아웃 (Header, AuthProvider)
│   ├── page.tsx            # 메인 - 인기글 30개
│   ├── board/              # 게시판 목록
│   ├── search/             # 검색 결과
│   ├── login/              # 로그인/회원가입
│   ├── admin/              # 관리자 카테고리
│   └── posts/
│       ├── new/            # 글쓰기
│       └── [id]/           # 상세, edit
├── components/
│   └── Header.tsx          # 공통 헤더
└── lib/
    ├── api.ts              # API 클라이언트
    └── auth.tsx            # 인증 Context
```

## 주요 모듈

### lib/api.ts
- `api<T>()`: JSON API 호출 (fetch 래퍼)
- `apiFormData()`: FormData 업로드 (이미지 등)
- `getToken()`: localStorage에서 JWT 조회
- `UPLOAD_BASE`: 이미지 URL 베이스

### lib/auth.tsx
- `AuthProvider`: 로그인 상태, JWT 저장
- `useAuth()`: login, register, logout, user
- localStorage: mini-board-token, mini-board-refresh

### Header
- 좌측: mini-board 로고, 게시판 링크
- 중앙: 검색 폼 → /search?q=
- 우측: 로그인/사용자정보/관리자/로그아웃

## API 연동
- 기본 URL: `NEXT_PUBLIC_API_URL` 또는 http://localhost:3001
- 인증: Authorization: Bearer {token}
