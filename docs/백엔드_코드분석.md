# Mini-Board 백엔드 코드 분석

## 기술 스택
- NestJS 11
- TypeORM + MySQL
- JWT (Access + Refresh)
- Elasticsearch (검색)
- Multer (파일 업로드)

## 디렉터리 구조

```
backend/src/
├── auth/           # 인증 (회원가입, 로그인, JWT)
├── users/          # 사용자
├── posts/          # 게시글 CRUD, 좋아요
├── comments/       # 댓글 (5 depth)
├── categories/     # 카테고리 (관리자)
├── search/         # Elasticsearch 검색
├── database/       # DB 설정, 시드
└── config/
```

## API 엔드포인트

| 경로 | 메서드 | 설명 |
|------|--------|------|
| /auth/register | POST | 회원가입 |
| /auth/login | POST | 로그인 |
| /auth/refresh | POST | 토큰 갱신 |
| /auth/logout | POST | 로그아웃 |
| /users/me | GET | 내 정보 |
| /posts/popular | GET | 인기글 30 |
| /posts | GET | 목록 (페이징, 검색) |
| /posts | POST | 작성 |
| /posts/:id | GET/PATCH/DELETE | 상세/수정/삭제 |
| /posts/:id/like | POST | 좋아요 |
| /posts/:id/comments | GET/POST | 댓글 |
| /comments/:id | PATCH/DELETE | 댓글 수정/삭제 |
| /search | GET | 헤더 검색 |
| /categories | GET/POST/PATCH/DELETE | 카테고리 |

## 인증
- JwtAuthGuard: 전역 적용, @Public() 데코레이터로 제외
- RolesGuard: @Roles(UserRole.ADMIN) 관리자 전용
