# 실시간 인기키워드 순위 로직

## 0. 비정상 점수(수조 단위) 발생 시

**과거 구현**에서 `ZADD` + `Date.now()`(타임스탬프)를 점수로 사용했음.  
이 데이터가 Redis에 남아 있으면, `ZINCRBY`로 1씩 더해도 기존 점수(수조)에 비해 변화가 거의 없어 순위가 바뀌지 않음.

**해결**: 관리자 > 통계 > "데이터 초기화" 버튼으로 Redis `keywords:rank` 키를 삭제 후 재시작.

---

## 1. 전체 흐름

```
검색 API 호출 → addKeyword() → Redis ZSET 저장 → broadcastKeywordsUpdate() → WebSocket 전송 → 프론트 UI 갱신
```

## 2. 코드별 동작

### 2.1 addKeyword (keywords.service.ts)

```typescript
await client.zincrby(KEYWORDS_KEY, 1, k);  // ① 검색어 점수 +1
await client.zremrangebyrank(KEYWORDS_KEY, 0, -(MAX_KEYWORDS + 1));  // ② 하위 키워드 제거
```

**① ZINCRBY**: Redis Sorted Set에서 해당 키워드의 점수에 1을 더함. 없으면 score=1로 추가.
- 예: "nest" 검색 → "nest" score +1 (최초면 1)

**② ZREMRANGEBYRANK**: rank 0 ~ -(MAX_KEYWORDS+1) 구간의 멤버 제거.
- Redis rank: 0 = 가장 낮은 점수, -1 = 가장 높은 점수
- rank 0 ~ -101 → **가장 낮은 점수부터 101번째까지** 제거 → 상위 100개만 유지

### 2.2 getTopKeywords (keywords.service.ts)

```typescript
const results = await client.zrevrange(KEYWORDS_KEY, 0, limit - 1);
```

**ZREVRANGE**: 점수 높은 순(내림차순)으로 0~4번째 반환 → 상위 5개

### 2.3 SearchController 연동

```typescript
if (q?.trim()) {
  void this.keywordsService.addKeyword(q.trim()).then(() => {
    void this.keywordsGateway.broadcastKeywordsUpdate();
  });
}
```

검색 시 addKeyword 완료 후 broadcast → WebSocket으로 모든 클라이언트에 갱신 전달.

## 3. 신규 키워드가 순위에 안 올라가는 이유

### 문제 시나리오

1. Redis에 이미 100개 키워드가 있음 (모두 score ≥ 2)
2. 신규 키워드 "newword" 검색 → ZINCRBY로 score=1 추가 → 총 101개
3. ZREMRANGEBYRANK 0 -101 실행 → **가장 낮은 점수 1개 제거**
4. "newword"(score=1)가 유일한 최저 → **즉시 제거됨**
5. 결과: 신규 키워드는 한 번 추가되자마자 trim에서 제거되어 순위에 반영되지 않음

### 핵심 원인

- 상위 100개만 유지하는 trim이 **항상** 실행됨
- 신규 키워드는 score=1로 들어오므로, 기존 100개가 모두 score≥1이면 **항상 최하위**가 됨
- 따라서 trim 시 **항상 신규 키워드가 먼저 제거**됨

---

## 4. 수정 방향

**trim 제거**: 키워드 개수 제한을 두지 않고 ZSET을 그대로 유지.

- 키워드 1만 개도 수 KB 수준으로 메모리 부담 거의 없음
- 신규 키워드가 ZSET에 남아 점수를 쌓을 수 있음
- 검색 횟수가 많은 키워드가 자연스럽게 상위로 올라감
